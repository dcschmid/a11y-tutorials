Die Bandbreite der Möglichkeiten von automatisierten Prüfwerkzeugen ist groß.
Einfache Tools testen eine Seite nach der anderen anhand eines festgelegten
Regelwerks. Einige können sogar nur auf eine ganz bestimmte Art von Fehlern
testen. Am anderen Ende des Spektrums gibt es Tools für automatisierte Tests
auf Unternehmensebene, die mit Hilfe von Spidern mehrere Seiten crawlen und
analysieren und die Möglichkeit haben, einen benutzerdefinierten Satz von
Prüfregeln auszuwählen, Skripte aufzuzeichnen, Formulare automatisch auszufüllen,
Wiederholungstests zu planen, Berichte zu automatisieren und vieles mehr.

Abgesehen von den Tools auf Unternehmensebene können grundlegende automatisierte
Testtools Erweiterungen oder Add-ons für einen bestimmten Browser sein, während
andere browserunabhängig sind. Mozilla Firefox und Google Chrome bieten eine Reihe
hilfreicher automatisierter Prüfwerkzeuge in Form von Add-ons oder Erweiterungen,
die mit wenig Aufwand in deinem aktuellen Browser installiert werden können. Im
Folgenden listen wir viele der Tools auf, die über Mozilla Firefox und Google
Chrome verfügbar sind, sowie einige browserunabhängige Tools:

## axe DevTools Browser-Erweiterung für Mozilla Firefox und Google Chrome

Die axe DevTools-Erweiterung ist eine leichtgewichtige JavaScript-Bibliothek, die
es Testern und Entwicklern ermöglicht, Inhalte im Browser zu bewerten. Dieses
leistungsstarke Tool verwendet Regeln zur Barrierefreiheit, die konsistente Ergebnisse
liefern, so dass Entwickler/innen einfach Korrekturen vornehmen können. Die Ergebnisse
liefern auch keine falsch-positiven Ergebnisse. Es ist eines der robustesten
kostenlosen Tools, die derzeit für automatisierte Tests der Barrierefreiheit
verfügbar sind. Dies ist das offizielle Tool, das Deque für seinen Testansatz
verwendet und das wir auf der nächsten Seite genauer beschreiben werden.

Einige der wichtigsten Funktionen sind:

- Bereitstellung einer umfassenden Liste der zu behebenden Probleme in einem
  einzigen, detaillierten Bericht

- Unterstützende Farbkontrastanalyse

- Interaktive Fehlerbehebung

- Erneutes Testen von Problemen

- Tool: <a href="http://www.deque.com/products/axe/" target="_blank">Informationen
  über axe DevTools Extension</a>

## Tools für Mozilla Firefox

### Vollständige Firefox-Tools zum Testen der Barrierefreiheit

#### Firefox Web Developer Toolbar

Diese Erweiterung enthält Dutzende von vorkonfigurierten Tests, mit denen du
Barrierefreiheit überprüfen kannst.

- Tool: <a href="https://addons.mozilla.org/en-US/firefox/addon/web-developer/"
  target="_blank">Webentwickler-Symbolleiste für Mozilla Firefox</a>

### Spezialisierte Firefox-Tools zum Testen der Barrierefreiheit

#### Firefox-Entwickler-Tools

Die früher verfügbare Firebug-Erweiterung, die ein allgemeines Entwicklungs- und
Debugging-Tool war, wurde direkt in die nativen Firefox-Entwicklerwerkzeuge
integriert, die es dir ermöglichen, dich wirklich auf bestimmte Codeschnipsel zu
konzentrieren und sie zu analysieren. Beim Surfen hast du eine Fülle von
Entwicklungswerkzeugen zur Hand. Du kannst CSS, HTML und JavaScript auf jeder
Webseite live bearbeiten, debuggen und überwachen.

- Gleichwertige Funktionen wie in Firebug finden

- Artikel: <a href='https://developer.mozilla.org/en-US/docs/Tools/Migrating_from_Firebug'
  target='_blank'>Migration von Firebug zu den Firefox-Entwicklerwerkzeugen - Wo
  du Funktionen findest, die in Firebug enthalten waren</a>

#### NoCoffee Vision Simulator

Der NoCoffee Vision Simulator ist eine Firefox-Erweiterung, mit der verschiedene
Arten von Sehbehinderungen simuliert werden können, um zu verstehen, welche
Probleme Menschen mit Sehbehinderungen beim Betrachten einer Website haben können.
Manche Websites verwenden kontrastarme Farben und kleine Texte und Zielbereiche
können für Menschen mit Sehschwäche eine Barriere darstellen. Tester/innen und
Entwickler/innen können mit diesem Tool feststellen, ob Text und Farben auf einer
Webseite für verschiedene Arten von Sehbehinderungen geeignet sind.

- Tool: <a href="https://addons.mozilla.org/en-US/firefox/addon/nocoffee/"
  target="_blank">NoCoffee Vision Simulator für Mozilla Firefox</a>

## Tools für Google Chrome

### Barrierefreiheit in den Chrome DevTools

Die Funktionen der inzwischen abgeschafften Erweiterung Accessibility Developer
Tools wurden zu den eingebauten Chrome Developer Tools hinzugefügt; dazu gehören
eine Registerkarte für die Barrierefreiheit und eine Seitenleiste für die
Barrierefreiheit auf der Registerkarte Elemente.

### Web Developer Toolbar

Diese offizielle Portierung der Web Developer-Erweiterung von Firefox enthält
Dutzende von vorkonfigurierten Tests, mit denen Barrierefreiheit geprüft werden
kann.

- Tool: <a href="https://chrome.google.com/webstore/detail/web-developer/bfbameneiokkgbdmiekhjnmfkcnldhhm"
  target="_blank">Web Developer Toolbar für Google Chrome</a>

### Firebug Lite für Google Chrome

Firebug Lite war die offizielle Portierung der Firefox-Erweiterung. Da sie nicht
mehr mit den neuesten Versionen von Chrome kompatibel ist, wurde sie von einem
anderen Autor geforkt und wieder verfügbar gemacht. Firebug Lite ist kein Ersatz
für Firebug oder die Chrome Developer Tools. Es ist ein Werkzeug, das in Verbindung
mit diesen Tools verwendet werden kann. Firebug Lite bietet die reichhaltige
visuelle Darstellung, die wir von Firebug gewohnt sind, wenn es um HTML-Elemente,
DOM-Elemente und Box-Modell-Schattierungen geht. Außerdem bietet es einige coole
Funktionen wie die Inspektion von HTML-Elementen mit der Maus und die
Live-Bearbeitung von CSS-Eigenschaften.

- Tool: <a href='https://chrome.google.com/webstore/detail/firebug-lite-for-google-c/ehemiojjcpldeipjhjkepfdaohajpbdo'
  target='_blank'>Firebug Lite für Google Chrome</a>

## Browserunabhängige Tools

Diese Tools sind nicht an einen bestimmten Browser gebunden:

### Color Oracle

Menschen, die farbenblind sind, reagieren empfindlich auf bestimmte Farben wie
Rot- und Grüntöne. Color Oracle ist ein Tool, mit dem du Webseiten-Designs auf
den effektiven Einsatz von Farben testen kannst. Color Oracle ist ein kostenloser
Farbenblindheitssimulator, der für Windows, Mac oder Linux verwendet werden kann.
Er wendet einen Farbfilter auf die Webseite an, der simuliert, wie eine Person
mit Farbenblindheit das Design sehen könnte. Tester/innen und Designer/innen
können mit diesem Tool feststellen, ob die im Design verwendeten Farben für
Menschen mit Farbenblindheit Probleme verursachen könnten.

- Tool: <a href="http://www.colororacle.org/" target="_blank">Color Oracle</a>

### Colour Contrast Analyser

Dies ist ein großartiges Tool für die Barrierefreiheit, wenn du durch Pixelpicking
das Farbkontrastverhältnis testen musst. Es simuliert auch bestimmte Sehbehinderungen,
einschließlich Farbenblindheit. Der CCA ist für Windows und Mac und in mehreren
Sprachen erhältlich, darunter Englisch, Französisch, Niederländisch, Italienisch,
Deutsch, Hindi, Koreanisch und traditionelles Chinesisch.

- Tool: <a href="" target="_blank">Colour Contrast Analyser</a>

### HTML_CodeSniffer

HTML_CodeSniffer ist ein clientseitiges Skript, das den HTML-Quellcode überprüft
und Verstöße gegen einen definierten Kodierungsstandard aufdeckt. HTML_CodeSniffer
ist vollständig in JavaScript geschrieben, benötigt keine serverseitige Verarbeitung
und kann von Entwicklern erweitert werden, um eigene Codierungsstandards durchzusetzen,
indem sie eigene "Sniffs" erstellen. Das Bookmarklet wurde für die Verwendung mit
Google Chrome, Firefox 10+, Safari 5+ und Internet Explorer 8+ entwickelt.

- Tool: <a href="http://squizlabs.github.io/HTML_CodeSniffer/"
  target="_blank">HTML_CodeSniffer</a>

### WAVE

WAVE ist ein kostenloser Onlinedienst, der die Barrierefreiheit einer Webseite durch
visuelle Darstellungen prüft. Er ist auch als Firefox- und Chrome-Erweiterung und
als API verfügbar.

- Tool: <a href="http://wave.webaim.org/" target="_blank">WAVE</a>

## Andere Tools

### Xcode Inspector für Barrierefreiheit (Mac)

Mit dem Accessibility Inspector, der in der Xcode App auf dem Mac verfügbar ist,
können Entwickler/innen Webanwendungen auf Barrierefreiheit testen. Mit dem Tool
können Entwickler sehen, wie ein Screenreader, insbesondere VoiceOver, mit
Steuerelementen und anderen Oberflächenelementen auf einer Webseite interagieren
kann. Entwickler können mit dem Tool die Barrierefreiheit ihrer Anwendungen
analysieren, z. B. das Label, den Wert und die Eigenschaften der einzelnen
Steuerelemente. Häufig wird das Tool in Verbindung mit VoiceOver verwendet,
um Seiten und Apps auf Barrierefreiheit zu testen und zu debuggen.

- Tool: <a href='https://itunes.apple.com/us/app/xcode/id497799835?mt=12' target='_blank'>Xcode</a>

- Tool: <a href='https://developer.apple.com/library/mac/documentation/Accessibility/Conceptual/AccessibilityMacOSX/OSXAXTestingApps.html'
  target='_blank'>Testen auf Barrierefreiheit unter OS X</a>

### Barrierefreiheit Viewer (Windows)

Der Accessibility Viewer (aViewer), der von der Paciello Group entwickelt wurde,
ist ein kostenloses, quelloffenes Prüfwerkzeug, mit dem Tester und Entwickler
ganz einfach auf die Barrierefreiheits-API-Informationen zugreifen können, die
von Browsern und unterstützenden Technologien wie Screenreadern erzeugt werden.
Ähnlich wie der Xcode Accessibility Inspector können Entwickler mit diesem Tool
Barrierefreiheitsprobleme von Webanwendungen testen und beheben.

- Tool: <a href='https://github.com/ThePacielloGroup/aviewer/releases'
  target='_blank'>Accessibility Viewer (aViewer)</a>
