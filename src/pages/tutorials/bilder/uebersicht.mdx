Blinde Menschen können nicht direkt auf Bilder zugreifen. Sie hören zu, wie ihre Bildschirmlesegeräte
den alternativen Text vorlesen, den der Entwickler oder Inhaltsersteller für das Bild bereitgestellt hat.
Bei taubblinden Nutzern wird der Text in aktualisierbare Braille-Zeichen umgewandelt, die sie mit ihren
Fingern ertasten.

Die Techniken für die Bereitstellung von Alternativtext variieren, je nachdem, ob das Bild im `<img>`-Element,
über SVG-Markup, über das HTML 5 `<canvas>`-Element oder über eine andere Methode dargestellt wird. Das Konzept
ist jedoch dasselbe: Der alternative Text muss aussagekräftig sein und das Bild auf eine Weise ersetzen, die
für blinde Nutzer sinnvoll ist.

Auch andere Arten von Inhalten erfordern in der Regel alternative Darstellungen, wie z. B.:

- Video (für das Untertitel und eine Abschrift erforderlich sind und das möglicherweise eine
  Audiobeschreibung benötigt)

- Audio (für das mindestens eine Abschrift erforderlich ist)

- Plug-ins (die eine effektive Nutzung ihrer jeweiligen APIs für die Barrierefreiheit erfordern)

- Dokumente (z. B. Word, PowerPoint, PDF usw., die ebenfalls eine effektive Nutzung der jeweiligen
  APIs erfordern)

Oft ist es am besten, auch eine HTML-basierte Alternativversion von Nicht-HTML-Inhalten zu erstellen, denn
HTML bietet in der Regel das größte Potenzial für Barrierefreiheit auf den unterschiedlichsten Geräten.
