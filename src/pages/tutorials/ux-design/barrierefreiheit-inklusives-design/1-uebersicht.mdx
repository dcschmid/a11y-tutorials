## Inklusives Design

Eine Designmethode, die darauf abzielt, die gesamte Bandbreite menschlicher Vielfalt, einschließlich eines breiten Spektrums von Fähigkeiten und Behinderungen, zu ermöglichen und zu berücksichtigen. Das Hauptziel besteht darin, einen einheitlichen Designansatz zu schaffen, der mehrere Methoden für den Zugriff auf dieselbe Funktionalität ermöglicht.

## Barrierefreiheit im Web

Die Eigenschaften, die ein Web-Erlebnis für eine möglichst große Gruppe von Nutzern zugänglich machen. Barrierefreiheit im Web bezieht sich auf das Endergebnis eines integrativen Designprozesses.

## Benutzerfreundlichkeit

Die Eigenschaften, die ein Web-Erlebnis intuitiv und einfach zu nutzen machen. Ein benutzerfreundliches Webdesign ist auf den Zweck ausgerichtet, für den die Website erstellt wurde.

Um die Definition der Barrierefreiheit im Web etwas ausführlicher zu gestalten, werden in den Web Content Accessibility Guidelines (WCAG) 2.1 vier Hauptprinzipien der Barrierefreiheit im Web aufgeführt:

- Wahrnehmbar - Informationen und Komponenten der Benutzerschnittstelle müssen für die Benutzer auf eine Weise darstellbar sein, die sie wahrnehmen können.
- Bedienbar - Die Komponenten der Benutzeroberfläche und die Navigation müssen bedienbar sein.
- Verständlich - Informationen und die Bedienung der Benutzeroberfläche müssen verständlich sein.
- Robustheit - Die Inhalte müssen so robust sein, dass sie von einer Vielzahl von Benutzer-Agenten, einschließlich assistiver Technologien, zuverlässig interpretiert werden können.
